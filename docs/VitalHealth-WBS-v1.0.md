# Work Breakdown Structure (WBS): VITAL Health - Android (v1.0)

## 1.0 Initiation
- **1.1 Project Kickoff:** Team alignment and goals set.
- **1.2 Stakeholder Analysis:** Roles and responsibilities defined.
- **1.3 Project Charter Execution:** Scope and constraints finalized.

## 2.0 Planning
- **2.1 Requirements Gathering:** Specific features and user stories.
- **2.2 Design System:** Color palette, typography, and component library.
- **2.3 Technical Architecture:** Room DB design, Hilt module setup, Supabase sync logic.

## 3.0 Execution
- **3.1 Authentication & Onboarding:**
    - 3.1.1 Email/Password Auth (Supabase).
    - 3.1.2 4-Step Onboarding Flow (Jetpack Compose).
- **3.2 Health Tracking Modules:**
    - 3.2.1 Weight Logging & Bars charts.
    - 3.2.2 Blood Pressure (Systolic/Diastolic) & Badges.
    - 3.2.3 Heart Rate Monitoring & Analytics.
    - 3.2.4 Mood Emoji Picker & Notes.
- **3.3 Medication Management:**
    - 3.3.1 Prescription Entry UI.
    - 3.3.2 Daily/Weekly Taking Reminders.
    - 3.3.3 Medication Refill Countdown logic.
- **3.4 Trends & Analytics:**
    - 3.4.1 Date segment filtering (Week/Month/Year).
    - 3.4.2 PDF Clinical Report Generation.
- **3.5 Journaling System:**
    - 3.5.1 Daily chronological timeline.

## 4.0 Monitoring & Control
- **4.1 Quality Assurance:** Unit tests, instrumented tests, and manual UAT.
- **4.2 Performance Monitoring:** Monitoring app weight, framerate, and boot times.
- **4.3 Sprint Reviews:** Bi-weekly demos of progress.

## 5.0 Closure
- **5.1 Final App Submission:** Google Play Store deployment.
- **5.2 Lessons Learned:** Retrospective on project process and outcomes.
- **5.3 Project Handover:** Transferring credentials and maintenance guidelines.
