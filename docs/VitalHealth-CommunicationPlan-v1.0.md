# Communication Plan: VITAL Health - Android (v1.0)

## 1. Introduction
This document defines the communication requirements for the project, ensuring that the right stakeholders receive the right information at the right time.

## 2. Stakeholder Matrix
| Stakeholder | Role | Information Needed | Method | Frequency |
| :--- | :--- | :--- | :--- | :--- |
| **Project Sponsor** | Approval of milestones | High-level status, budget updates | Email / Status Report | Monthly / As needed |
| **Team Leads** | Direction on features | Technical specifications, roadblocks | Slack / Daily Stand-up | Daily |
| **Developers/Designers** | Clear task requirements | Jira tickets, PR reviews | Github / Slack | Ongoing |
| **Project Manager** | Coordination/Execution | Progress vs schedule, risk updates | Progress Meetings | Weekly |

## 3. Communication Channels
- **Slack:** Real-time messaging for immediate issues and daily updates.
- **Github:** Version control, pull request discussions, and technical issue tracking.
- **Jira:** Feature tracking, bug reporting, and sprint management.
- **Zoom/Meet:** Synchronous video conferencing for deep-dive discussions and demos.
- **Email:** Formal communication, document distribution, and major approvals.

## 4. Recurring Meetings
| Meeting | Goals | Participants |
| :--- | :--- | :--- |
| **Daily Stand-up** | Yesterday, Today, Blockers | Core dev team, PM |
| **Weekly Sync** | Weekly progress, roadmap check | PM, Team Leads |
| **Sprint Review** | Demo completed work | Entire project team |
| **Project Retrospective** | Reflect on what went well/poorly | Entire project team |

## 5. Document Management
All project documentation will be stored centrally in the `docs` folder of the repository. Naming convention: `VitalHealth-DocType-vX.X.md`.
